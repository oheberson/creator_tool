## Architecture & Implementation Plan (MVP → Production-Ready)

**Project Goal**  
A helper-first webapp that guides aspiring YouTubers through research → idea refinement → script → malleable timeline → export (or assisted in-app editing), with AI as coach, never as replacement. Philosophy: user always owns the final creative decisions.

**Date**: April 2026  
**Target Stack** (as requested): Next.js (App Router) + TypeScript + React + Supabase + modern React libs for smooth UX.

### 1. High-Level Architecture Overview
- **Frontend**: Next.js 15+ (React 19, Server Components where possible)
- **UI Framework**: Tailwind + shadcn/ui + Framer Motion (smooth interactions)
- **Node-Based Canvas / Workflow**: xyflow (React Flow) — battle-tested for infinite canvas + nodes (n8n/Figma-style). Nodes represent agents/steps; edges show dependencies. Supports custom nodes, real-time updates, and drag-and-drop.
- **Timeline Editor**: Custom component built on React Flow + Twick SDK (or pure React + WebCodecs/FFmpeg.wasm for MVP). Drag-and-drop clips, placeholders, timing, B-roll suggestions.
- **Agent Orchestration**: LangGraph.js (stateful graphs) — perfect for revisitable, human-in-the-loop workflows. Each node = an agent step with checkpointing.
- **Backend / DB**: Supabase (PostgreSQL + pgvector for RAG, Auth, Realtime, Storage for user assets)
- **AI Layer**: Vercel AI SDK + Anthropic Claude 4 / OpenAI GPT-4o / Gemini (via API keys). Multi-modal for video/image analysis.
- **Video Processing**: Browser-first with FFmpeg.wasm + WebCodecs (zero server cost for MVP). Optional hybrid cloud (Mux/Shotstack) for heavy renders in paid tier.
- **Hosting**: Vercel (frontend + serverless functions) + Supabase.

React Flow (xyflow): Most mature node-based library in 2026; infinite canvas, custom nodes, edges, and real-time updates out-of-the-box.

Browser video processing (FFmpeg.wasm + WebCodecs): Zero infra cost for MVP; privacy-first; aligns with “helper” ethos (user files stay on device until they choose).
Supabase + pgvector: Instant auth, realtime, vector search for RAG — no extra services needed initially.
No heavy backend: Everything serverless until video rendering scales (then add optional cloud workers).
